# DABOM-FE Code Review Style Guide

리뷰는 한국어로 작성합니다. 피드백은 명확하고 구체적이며, 원인과 개선안을 함께 제시합니다.

## 프로젝트 컨텍스트

- **모바일 퍼스트 PWA** 웹앱 (Next.js Standalone)
- Turborepo 모노레포 + pnpm 워크스페이스
- React 19 + TypeScript + Next.js 16 + Tailwind CSS v4
- Three.js (@react-three/fiber) 기반 3D 렌더링 포함
- 역할(Role) 중심 Tailwind 컬러 토큰 사용 (tailwind.config.ts에서 관리)
- 모노레포 구조: apps/service (사용자), apps/admin (관리자), packages/shared (공유 라이브러리)

## 1) 코드 품질

- 변수/함수/컴포넌트 네이밍이 역할을 명확히 드러내는지 점검
- 매직넘버/문자열은 상수 또는 enum으로 분리
- 새로운 기능 추가나 수정 시, 기존 코드와의 호환성·재사용성뿐 아니라 확장 가능한 구조인지 검토

## 2) React/TypeScript 베스트 프랙티스

- React/TypeScript 최신 베스트 프랙티스를 우선 적용
- 컴포넌트는 단일 책임 원칙(SRP)을 따르는지 확인
- 불필요한 리렌더링이 발생하지 않는지 점검 (메모이제이션, key 등)
- `any` 타입 사용을 지양하고 구체적인 타입 정의를 권장

## 3) 가독성과 안정성

- 로직이 단순하고 직관적으로 읽히는지 확인
- 에러/예외 처리가 적절한지 최우선으로 검토 (API 호출, 비동기 처리 등)
- 복잡한 비즈니스 로직, 훅, 상태 관리는 JSDoc이나 간단한 주석을 권장

## 4) 접근성과 호환성

- 브라우저/디바이스 호환성 문제가 없는지 확인
- 접근성(ARIA 속성, 키보드 내비게이션, 시맨틱 HTML) 점검
- 이미지에 alt 텍스트, 폼 요소에 label이 있는지 확인

## 5) PWA

- Service Worker 등록/업데이트 로직이 올바른지 확인
- 오프라인 캐시 전략이 적절한지 검토 (네트워크 우선 vs 캐시 우선)
- Web App Manifest 변경 시 아이콘, theme_color, display 모드 정합성 점검

## 6) 모바일 퍼스트 & 반응형

- 모바일 화면(375px 기준)에서 먼저 동작하는지 확인
- 터치 타겟 크기 최소 44×44px 확보
- 뷰포트 단위(dvh, svh) 사용 시 브라우저 호환성 점검
- 스크롤/제스처 동작이 모바일에서 자연스러운지 확인

## 7) Three.js / 3D 렌더링

- Canvas/WebGL 리소스 해제 (dispose) 가 적절히 이루어지는지 확인
- 3D 씬의 렌더 루프가 불필요하게 돌지 않는지 점검 (frameloop 설정)
- 모바일 기기 성능을 고려한 폴리곤 수/텍스처 크기 확인
- useFrame, useLoader 등 R3F 훅 사용이 올바른지 검토

## 8) 스타일링 (Tailwind 컬러 토큰)

- 직접 색상 값(#fff, rgb 등) 대신 tailwind.config.ts에 정의된 역할 기반 토큰 사용 여부 확인
- 새 색상이 필요할 경우 토큰으로 등록 후 사용하도록 안내
- Tailwind 클래스 순서가 일관적인지 점검 (eslint-plugin-tailwindcss)

## 9) 테스트

- 단위 테스트/통합 테스트가 충분히 작성되어 있는지 검토
- 테스트하기 쉬운 구조인지 (의존성 분리, 순수 함수 등) 확인
- 신규 로직은 최소 단위 테스트 1개 이상 권장

## 10) 리뷰 코멘트 스타일

- "왜 문제인지" + "어떻게 고치면 좋은지"를 함께 제시
- 필요 시 예시 코드도 포함
- 리뷰는 꼭 필요한 부분에 집중하되, 전체 맥락 이해를 돕기 위해 추가 설명은 요약으로 제공
